Stochastic Synthesis of Complex Sounds

for Game Gems III


(C) 2000 Phil Burk, http://www.softsynth.com/

This C++ software was written to accompany two chapters in the book Game Gems III.

Running the Demo

An executable file have been provided for computers running Windows.

This demonstrate the use of stochastic (random) synthesis to generate several complex sounds. To hear the demo, double click on "StochasticDemo.exe".

Hit the '?' key to see the menu. You can turn the various sounds on and off. For the rain, you can increase the number of raindrops per second by hitting 'R' and decrease the rate by hitting 'r'.

Compiling the Demo

In the win32 folder, you will find a GameGems.dsw Visual C++ workspace. Double click on that and then compile the project. The executable will be placed in the win32 folder.

These demos use the PortAudio library for audio I/O. This is a free open-source cross platform audio API that runs on Windows, Macintosh and Unix. You should be able to compile the software with little or no modification for those platforms. I have provided source code for the Windows MME implementation. For serious development, or to compile on non-windows platforms, you should download the latest PortAudio source code from:

    http://www.portaudio.com/

The PortAudio web site has documentation and tutorials that explain how to use it on diffeent platforms.

Software Design

The software is based on a unit generator architecture. Each unit generator creates or processes sound. The output of a unit can be pass to another for processing. Complex circuits can be built from smaller units or circuits.

The units and circuits have a gen() method that generates the outputs samples using whatever function is specific to the unit. A Unit_Add, for example, simply adds its inputs together to make an output.

The audio synthesis is normally done in small blocks instead of one sample at a time. The output samples are written to the outputs array. This is more efficient for the CPU. Some units, however, create very slow signals and only need to be generate one sample per block. They have a gen1() method that only generates one sample and writes it to outputs[0].

The calculations are done using 32 bit float data types defined as GGSynthFloat. This is faster than doubles and is plenty accurate enough for game synthesis.
